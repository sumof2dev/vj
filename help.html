🌐 Connection Workflows

Option 1: Global "Secret Code" (Zero-Config)

This is the intended professional workflow. You set up a Cloudflare Tunnel on the device, and anyone can control it from anywhere using just your Secret Code.

  1. Run ./setup_tunnel.sh on the device.
  2. Follow the prompts to create your code (e.g. mybox).
  3. Go to ravebox.love and enter your code.
  4. Result: Instant, secure, no "Failed to Fetch" errors.

Option 2: Local Network (Manual)

Access the device directly via its local IP address. Since we use self-signed certificates for security, the browser will block requests until you "Sign Off" manually.

  • Access: https://[LOCAL-IP]:8000/manager.html
  • Browsers will show "Your connection is not private".
  • Click Advanced -> Proceed.
  • If "Failed to Fetch" appears in the UI, look for the "Authorize Port" helper at the bottom and follow its links.

How the RaveBox Raves

The engine uses a sophisticated multi-stage pipeline to turn raw sound into coordinated light shows.

⚡ The Audio Pipeline

The engine captures system audio using PulseAudio and performs real-time analysis:

  • 6 Custom Frequency Bins: Categorized as Sub+Bass, Low-Mid, Mid, High-Mid, Presence, and Air/High.
  • Spectral Flux Tracking: Detects physical "impact" and transient energy shifts rather than just simple volume peaks.
  • Normalization: A rolling 5-second window maps variable inputs to a perfect 0.0-1.0 range, ensuring the show stays "full" even during quiet track sections.

🧠 EDM Intelligence

RaveBox understands track structure by tracking musical "Transients":

  • Building: Detected when cumulative energy is rising fast.
  • Tension: The "Pre-Drop" state where volume/energy is high but bass vanishes.
  • Dropping: The massive impact state that triggers high-intensity patterns.

🎨 Layered DMX Logic

Commands pass through a 4-layer priority stack before reaching the fixtures:

  1. Spatial & Frequency Logic: LFOs (Sine, Tri, Saw) or Direct Frequency Mapping (pinpointing the most dominant EQ bin) mapped to your stage dimensions.
  2. Base Presets: Animated scenes like "Lissajous" or "Voxel Tunnel" provide the rhythmic foundation.
  3. System Triggers (High Priority): Preset-driven overlays (e.g., "Machine Gun" or "Wonky" presets) that automatically punch through when specific vibe, transient, or EQ conditions are met. These can even override global blackouts.
  4. Manual Overrides (Performance): Direct additive injections from the UI or physical Gamepads. Manual frequency control acts as a layer on top of the automated analysis.

Initial Setup

Advanced configuration for Raspberry Pi 5 hardware and system services.

1. Hardware Config

Edit /boot/firmware/config.txt to enable UART for RS485 and HifiBerry audio:

dtparam=uart0=on
dtoverlay=disable-bt
enable_uart=1
dtoverlay=uart0-pi5
dtoverlay=hifiberry-dacplus

2. System Services

The engine runs as three independent systemd units:

  • vj-server: The host for the Web UI and Assets (Management Console).
  • vj-launcher: The management API that handles engine startup/shutdown and configuration.
  • vj-engine: The heavy-lifter: Handles Audio Analysis, Vibe/Transient Detection, and the DMX Logic Engine.

Run ./setup_service.sh to re-install or update these services.

3. SSL & Mobile Standalone

To use the Remote view on mobile without browser chrome, you must enable SSL. Run ./generate_cert.sh then accept the risk once in your mobile browser. This unlocks the "Install App" PWA features.
